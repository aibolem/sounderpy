Playing with sounding settingsÔÉÅ

Great! We made two easy sounding plots, both of which required a single, simple line of code.

Now we can adjust a few settings to customize the plot a little bit.

To do so, let‚Äôs first study the build_sounding() function‚Ä¶

build_sounding(clean_data, style='full', color_blind=False, dark_mode=False, storm_motion='right_moving', special_parcels=None, save=False, filename='sounderpy_sounding')ÔÉÅ

where:

  • clean_data (dict, required) ‚Äì the dictionary of data to be plotted

  • style (str, optional) ‚Äì may be ‚Äòsimple‚Äô or ‚Äòfull‚Äô. Default is ‚Äòfull‚Äô.

  • color_blind (bool, optional) ‚Äì whether or not to change the dewpoint trace line from green to blue for improved readability for color deficient users/readers. Default is False

  • dark_mode (bool, optional) ‚Äì True will invert the color scheme for a ‚Äòdark-mode‚Äô sounding. Default is False.

  • storm_motion (str or list of floats, optional) ‚Äì the storm motion used for plotting and calculations. Default is ‚Äòright_moving‚Äô. Custom storm motions are accepted as a list of floats representing direction and speed. Ex: [270.0, 25.0] where ‚Äò270.0‚Äô is the direction in degrees and ‚Äò25.0‚Äô is the speed in kts.

  • special_parcels (nested list of two lists, optional) ‚Äì a nested list of special parcels from the ecape_parcels library. The nested list should be a list of two lists ([[a, b], [c, d]]) where the first list should include ‚Äòhighlight parcels‚Äô and second list should include ‚Äòbackground parcels‚Äô. Another option is ‚Äòsimple‚Äô, which removes all advanced parcels making the plot quicker.

  • save (bool, optional) ‚Äì whether to show the plot inline or save to a file. Default is False which displays the file inline.

  • filename (str, optional) ‚Äì the filename by which a file should be saved to if save = True. Default is ‚Äòsounderpy_sounding‚Äô.

Saving plots as a fileÔÉÅ

By default, SounderPy is set to show plots inline. If you want/need to save them to a file, SounderPy offers that option!

Just set the save kwarg to True, and give it a file name using the filename kwarg!

IMPORTANT NOTE!!!ÔÉÅ

  • If your plots come out with a funky layout when displaying them inline, it may be due to your screen resolution.

  • In the event that this is the case, simply use the save=True option to save plots to a file!
